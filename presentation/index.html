Loop 1 — Discovery

Understand before you build.

The double diamond: diverge to explore the problem space, converge to a brief, diverge to explore solutions, converge to a spec.

Quality gate
Are we solving the right problem?

Loop 2 — Volere

Specify what "done" means.

Two proven disciplines, combined for the first time. Volere (Robertson & Robertson, 1995) provides structured requirement types. The V-Model (systems engineering) maps each definition level to a verification level. This framework connects them.

Define (Volere) Verify (V-Model)
BUC (Business Use Case) Acceptance Tests
PUC (Product Use Case) System Tests
UR (User Requirement) System Tests
Architecture / Design Integration Tests
TC (Technical Constraint) Unit Tests
↓ Code & Static Analysis ↓
The synthesis
Volere defines what. The V-Model defines how to verify.
No published work had mapped these together for agentic development. This framework does.

One requirement. Multiple definitions of "done."

Traditional: one pass/fail. Volere: user + security + ops + regulatory.

user

operational
"Works during cross-machine deployment"
V-Model: system test (PUC level)
regulatory
"Meets IEC 61508 SIL-2 evidence chain"
V-Model: acceptance test (BUC level)

Not everything needs the same rigour.

Design Assurance Level (DAL) — borrowed from aerospace safety standards (DO-178C), adapted for software. Classifies each requirement by the impact of failure: if this breaks, how bad is it? The answer determines how much verification you invest.
